## ‚ö†Ô∏è Common Pitfalls to Avoid

### 1. **Directory Issues**
**Problem**: TinaCMS requires an empty directory for setup
**Solution**: Move existing files temporarily, then restore after setup
**Lesson**: Always check tool requirements before running setup commands

### 2. **Outdated CLI Flags**
**Problem**: AI might suggest flags that don't exist (`--skipTelemetry`)
**Solution**: Ask AI to check help documentation (`-h` flag)
**Lesson**: Verify CLI commands, especially for rapidly evolving tools

### 3. **Cache Problems**
**Problem**: Getting errors like "Unable to find record" after content changes
**Solution**: Delete `.next` and `.tina/__generated__` folders, restart dev server
**Lesson**: When TinaCMS acts weird, clear generated files and rebuild

### 4. **Color Consistency Issues**
**Problem**: Colors look different in light/dark mode
**Solution**: Use `oklch()` color space instead of RGB/HSL
**Lesson**: Modern color spaces solve old problems - learn new CSS features!

### 5. **Prose Styling Conflicts**
**Problem**: Dark mode text becomes invisible
**Solution**: Use `dark:prose-invert` and explicit color classes
**Lesson**: Tailwind's prose plugin needs explicit dark mode configuration

### 6. **Git Detached HEAD**
**Problem**: After checking out commits, in detached HEAD state
**Solution**: Checkout specific commit hash or create new branch
**Lesson**: Be careful with git operations during development

### 7. **Port Conflicts**
**Problem**: "Tina Dev server is already in use. Datalayer server is busy on port 9000"
**Solution**: Kill existing processes on ports 3000, 4001, 9000
**Lesson**: Always clean up dev servers before restarting

### 8. **File Reference Errors**
**Problem**: Blog post crashes because author file doesn't match
**Solution**: Ensure author paths match exactly (`content/authors/pedro.md`)
**Lesson**: TinaCMS uses exact file references - typos break things

## üí° General Suggestions for Vibe Coding

### Prompt Engineering Best Practices

**1. Set Context First**
```
Act as a [X] expert and use official [Y] documentation as reference
```
This primes AI to think like an expert and seek authoritative sources.

**2. Break Complex Tasks into Steps**
```
0. Create instructions.md to track progress
1. Set up project
2. Install dependencies
3. Start dev server
```
Numbered steps help AI understand sequence and dependencies.

**3. Specify Non-Interactive Requirements**
```
Generate the full command with proper flags so it runs 
non-interactively (no manual input needed)
```
Prevents AI from creating commands that pause for user input.

**4. Give Positive Reinforcement**
```
you are doing a great work. keep it up.
```
Helps maintain the AI's direction and approach.

**5. Preview Future Steps**
```
in the next step we will change the site theme to use red...
```
Allows AI to prepare and make compatible choices now.

**6. State What You Like**
```
I really like the styling
```
Tells AI what to preserve during changes.

**7. Be Specific About Problems**
```
The hero section subtitle looks very weird, One line for 
I cook with TinaCMS (the best CMS) and Next line I build 
great software at SSW.
```
Don't just say "fix it" - explain what's wrong and what you expect.

**8. Provide Options When Flexible**
```
either remove it or tell me where I can put in image
```
Gives AI room to suggest the best solution.

### Development Workflow Tips

**1. Track Progress**
- Use instructions.md or similar to document steps
- Makes it easy to resume if interrupted
- Creates documentation for future reference

**2. Iterate Quickly**
- Make changes ‚Üí See results ‚Üí Refine
- Don't try to perfect everything in one prompt
- Vibe coding is about rapid iteration

**3. Trust but Verify**
- AI is smart, but check critical outputs
- Verify CLI commands before running
- Test dark/light modes, responsive design

**4. Clear Cache When Stuck**
- Delete `.next` folder
- Remove `.tina/__generated__`
- Restart dev server fresh

**5. Commit Frequently**
- Small, focused commits
- Makes it easy to rollback if needed
- Tracks what each prompt accomplished

### Content Strategy

**1. Start with Structure**
- Navigation ‚Üí Hero ‚Üí Content ‚Üí Footer
- Get the skeleton right before styling

**2. Content Before Polish**
- Add your personal info first
- Then refine styling and colors
- Easier to see what works with real content

**3. Test Incrementally**
- Check after each major change
- Don't stack 10 changes and debug later
- Faster to fix issues immediately

## Step 2: Modern Tech Blog Aesthetic

This is where GitHub Copilot really shined! We transformed the default blog into a modern masterpiece.

### Color System Overhaul

We rebuilt the entire color system using `oklch()` color space for better consistency across light and dark modes:

```css
:root {
  /* Light mode - Clean white with red accent */
  --primary: oklch(0.55 0.22 25);
  --secondary: oklch(0.95 0.05 60);
  --background: oklch(0.99 0 0);
  --foreground: oklch(0.15 0 0);
}

.dark {
  /* Dark mode - Deep charcoal with vibrant accents */
  --primary: oklch(0.65 0.25 25);
  --secondary: oklch(0.25 0.03 60);
  --background: oklch(0.12 0 0);
  --foreground: oklch(0.95 0 0);
}
```

**Why oklch()?** It provides perceptually uniform colors that look consistent across different lightness levels - perfect for dark mode!

### Custom Utility Classes

We added custom CSS utilities for that modern tech blog feel:

```css
/* Gradient text effect */
.gradient-text {
  @apply bg-gradient-to-r from-primary via-purple-500 to-pink-500 
         bg-clip-text text-transparent;
}

/* Frosted glass effect */
.glass {
  @apply backdrop-blur-md bg-background/50 border border-border/50;
}

/* Card hover animation */
.card-hover {
  @apply transition-all duration-300 hover:shadow-lg 
         hover:shadow-primary/10 hover:-translate-y-1;
}
```
